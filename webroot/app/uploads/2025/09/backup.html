            <h2 class="hub-tagline__text">

                The Framework outlines consistent best practices guidance for sharing data between trusted parties 

    and a nationally consistent method for modifying data for public release.

    

            </h2>

          

            <p class="hub-intro__content-text">

                Restricted Access Species Data (RASD) are biodiversity datasets 

                which contain information which may compromise 

                individuals, species, personal property or

                landholdings and that have some restriction

                over availability of the data.

            </p>

                <p class="hub-intro__content-text">

                    Types of restricted access species data include

                    <a class="link" href="./pdf-html/index.html">(see the Framework for more detail)</a>:

                 <ul class="hub-intro__content-list">   

                <li>1. <b>Personal Identifiable Information</b></li>

                <li>2. <b>Indigenous Data</b></li>

                <li>3. <b>Usage-Restricted Categories including:</b></li>

                    <ul>

                        <li>3.1 Legal contract</li>

                        <li>3.2 Legal financial</li>

                        <li>3.3 Non-legal</li>

                    </ul>

                <li>4. <b>Species-Related Categories including:</b></li> 

                    <ul>

                        <li>4.1 Location-related data</li>

                        <li>4.2 Identification data</li>

                        <li>4.3 Attribute data</li>

                    </ul>

                </ul>

              </p>

    <div class="hub-about__container">

        <h3 class="header-3">ABOUT THE PROJECT</h3>

        

        <p class="hub-about__text">

        

        The <a class="link" href="./pdf/RASD-Project-Overview.pdf">Restricted Access Species Data Pathways project</a> (previously named Sensitive Species Data Pathways project) is an initiative supported by the Australian Research Data Commons (ARDC) and the Atlas of Living Australia (ALA).

        </p>


        <p class="hub-about__text">The RASD Pathways project was a collaborative effort between the ALA, ARDC, Australian Government, state and territory conservation agencies, museums, herbaria, EcoCommons and the Western Australian Biodiversity Science Institute. The project is part of Australia's research infrastructure and was designed to respond to multiple government and non-government calls to improve access to withheld or modified biodiversity data by streamlining processes and systems to enable the sharing of sensitive species (restricted access species) data for authorised and approved users. The three outputs from the project were:</p>


        <p class="hub-about__text">1. A <a class="link" href="./pdf-html/index.html">national framework</a> outlining consistent best practice guidance for sharing RASD between trusted parties and a nationally consistent method for modifying restricted access data for public release.</p>

            <p class="hub-about__text">2. A <a class="link" href="https://service.rasd.org.au/">Restricted Access Data Service (RASDS)</a> where users can request and access RASD from custodians via a single point of contact.</p>

                <p class="hub-about__text">3. A trial proof-of-concept trusted environment for using RASD for modelling and analysis, developed by <a class="link" href="https://www.ecocommons.org.au/">EcoCommons Australia</a>.</p>

        <p>For further information, see the <a href="./pdf/RASD-FAQs.pdf" class="link">FAQs</a> or contact us on <a href="mailto:info@rasd.org.au" class="link">info@rasd.org.au</a>.</p>
